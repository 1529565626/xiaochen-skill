# 通用引导式学习框架

> 一套**领域无关**的 AI 辅助学习方法。交互方式固定，知识内容可替换——用苏格拉底式对话 + 结构化进度追踪，把任何系统化知识学深学透。

本仓库的核心资产是 `LEARNING-FRAMEWORK.md`，附带一个已填充好的案例 `AI-Prompt-Study/`（AI 提示词工程学习），用于演示框架如何落地。

---

## 一、是什么

`LEARNING-FRAMEWORK.md` 是一份**可复用的学习框架模板**。它定义了一套完整的 AI 辅助自学模式：

```
学生提问 → AI 先问"你已知什么" → 基于已有知识构建新内容
        → 解释后追问验证 → 根据回答调整教学
        → 自动记录会话 + 更新进度追踪
```

**不变的部分（框架提供）：**

| 要素 | 内容 |
|------|------|
| 教学对话模式 | 苏格拉底式提问 → 讲解 → 理解检查 → 适应性跟进 |
| 仓库目录结构 | `sessions/` + `progress/` |
| 会话记录模板 | 统一的提问记录、知识差距、掌握主题格式 |
| 进度追踪维度 | 已掌握 / 知识差距 / 信心水平 / 优先级 |
| 行为约束协议 | 不猜测、必须搜索验证、引用来源 |

**可变的部分（你填入）：** 学习领域名称、知识模块划分、具体主题列表、权威参考来源、领域特定规则。

框架内所有占位符用 `【】` 标记，替换掉就是你的专属学习系统。

## 二、为什么有用

### 认知科学依据

| 原理 | 框架如何实现 |
|------|-------------|
| **检索练习** | 每次提问先追问"你已知什么"，强制大脑主动检索，增强记忆编码 |
| **精细加工** | 在已有知识基础上构建新知，建立新-旧神经关联 |
| **间隔重复** | 进度追踪器标记已掌握和薄弱点，自然形成间隔复习节奏 |
| **即时反馈** | 每次理解检查即时纠正误解，避免错误固化 |
| **元认知** | 学生自评信心水平，提升对自身知识状态的觉察力 |

### 解决的核心问题

- **自学无反馈** → AI 导师实时追问、验证、纠偏
- **进度不可见** → 唯一追踪文件告诉你"会了什么、不会什么"
- **学了就忘** → 掌握/差距标注天然驱动间隔复习
- **查资料难以启动** → 苏格拉底方法先锚定你现有知识，再填补缺口
- **不知道先学什么** → 模块权重 + 优先级排序给出明确路径

### 框架边界

- **适合**：有明确知识体系的系统化学习（考试、认证、学科、技能树）
- **不太适合**：纯身体技能训练、完全无结构的开放探索
- **前置条件**：学生愿意回答问题——苏格拉底方法依赖双向互动

## 三、怎么用 + 示例

本节以 `AI-Prompt-Study/` 为案例，展示如何将框架模板落地为一个真实的学习项目。

### 3.1 启动流程（8 步）

```
1. 复制 LEARNING-FRAMEWORK.md 到新目录
2. 填入领域信息：按占位符速查表替换所有 【】
3. 定义知识模块：将领域拆分为 5-10 个模块，按重要性排序
4. 列出主题清单：每个模块下列出 4-7 个具体主题（总数 30-80 个）
5. 分配权重：标注每个模块的百分比/优先级
6. 初始化进度追踪器：创建空追踪文件
7. 清空会话记录：删除示例会话，保留模板
8. 启动学习：打开 Claude Code，开始第一个问题
```

### 3.2 案例对照：框架 vs AI-Prompt-Study 落地

以下是框架中关键占位符在 AI-Prompt-Study 中的实际替换：

| 框架占位符 | AI-Prompt-Study 填入值 |
|-----------|----------------------|
| `【项目名称】` | `AI-Prompt-Study` |
| `【学习领域】` | `AI 提示词工程（Prompt Engineering）` |
| `【领域】` | `ai-prompt` |
| `【模块A名称】` | `提示词基础理论` |
| `【模块B名称】` | `提示词结构设计` |
| `【权威来源】` | `Anthropic 官方文档、OpenAI Cookbook、arXiv` |

**案例的知识模块划分（7 模块 / 41 主题）：**

```
A. 提示词基础理论（20%）      — 6 主题
B. 提示词结构设计（20%）      — 6 主题
C. 角色与上下文工程（15%）    — 5 主题
D. 推理引导技术（15%）        — 7 主题
E. 提示词优化与调试（10%）    — 5 主题
F. 提示词安全与防护（10%）    — 5 主题
G. 实战场景应用（10%）        — 7 主题
```

**案例的学习路径设计：**

```
第1-2周：A（基础理论）+ B（结构设计）同时推进
         ↓
第3-4周：C（角色与上下文）+ D（推理引导）
         ↓
第5-6周：E（优化调试）+ F（安全防护）
         ↓
持续：   G（实战贯穿全程，每周练习一个场景）
```

### 3.3 案例的 CLAUDE.md 教学流程（含实战演示）

打开 Claude Code 后，AI 会自动加载 `CLAUDE.md` 中的导师指令。以下是一次典型的教学交互：

**学生**："怎么写一个好的 system prompt？"

**Claude（初步探索）**：
> "好问题！在深入之前，让我先了解一下——你之前写过 system prompt 吗？对 system prompt 和 user prompt 的区别有什么理解？"

**学生**："我大概知道 system prompt 是设置 AI 行为的，但不太确定怎么写才有效。"

**Claude（讲解）**：
> "很好，让我在你的基础上补充。System prompt 的核心作用是**设定游戏的规则**——它定义 AI 在这个对话中的身份、行为边界和输出标准。一个好的 system prompt 通常包含三个关键要素：
>
> 1. **身份锚定**：你是什么角色？例如'你是一位精通 Python 的代码审查专家'
> 2. **行为约束**：你能做什么、不能做什么？
> 3. **输出规范**：回答应该长什么样？
>
> 一个常见错误是把所有信息都塞进 system prompt……"

**Claude（理解检查）**：
> "现在检验一下：如果让你给一个'帮助用户写周报'的场景设计 system prompt，你会怎么确定这三个要素？"

**会话结束后的自动动作：**
1. 创建 `/sessions/2026-05-21/session-notes.md`，记录完整的提问-回答-理解检查过程
2. 更新 `/progress/ai-prompt-study-tracker.md`：标记 C.15 已掌握，调整模块进度百分比

## 四、还可以用的五个场景

### 1. 职业资格认证考试

**领域示例**：CPA（注册会计师）、CFA（金融分析师）、PMP（项目管理）、法律职业资格、执业医师

**为什么适合**：这类考试有公开的考纲（知识模块天然划分完毕）、固定题型和明确的权重分布。框架的"模块→主题→优先级"结构直接映射考纲。

**关键适配**：权威来源替换为官方考纲和历年真题；验证协议中的"在线搜索"替换为查阅真题解析。

---

### 2. 编程语言与框架系统学习

**领域示例**：Rust 系统编程、React 生态、Kubernetes 运维、Go 后端开发

**为什么适合**：编程知识有清晰的层级关系（语法→标准库→设计模式→项目实战），适合拆分为模块。AI 导师可以做到传统教材做不到的：根据你的已有语言经验跳过冗余内容（比如已会 Python 的人学 Go 不需要从零讲控制流）。

**关键适配**：增加"动手练习"环节的权重；权威来源替换为官方文档和 RFC；进度追踪中为每个主题增加项目实战验证。

---

### 3. 学术课程/大学专业课

**领域示例**：数据结构与算法、操作系统、计算机网络、线性代数、机器学习

**为什么适合**：大学课程本来就有教学大纲和教材章节划分，直接映射到框架模块。AI 可以在你卡住时提供无限耐心的一对一辅导。

**关键适配**：CLAUDE.md 中增加公式、定理的验证协议；使用教材章节目录作为主题列表；权威来源替换为指定教材和论文。

---

### 4. 产品设计与用户体验方法论

**领域示例**：Design Thinking、用户研究方法、信息架构、可用性测试

**为什么适合**：设计方法论有成熟的知识体系（如 Nielsen 十大可用性原则、双钻模型），但传统学习方法缺乏互动反馈。AI 可以针对你正在做的真实项目给出即时的方法论应用建议。

**关键适配**：知识模块按"理论→方法→工具→案例"组织；增加"作品集评审"环节——学生上传自己的设计稿，AI 用方法论框架给予结构化反馈。

---

### 5. 企业入职培训与内部流程学习

**领域示例**：公司技术栈入门、合规流程、安全规范、内部工具链使用

**为什么适合**：企业培训材料通常是长篇文档，新人自学缺乏互动。框架将其转化为问答式对话，AI 扮演内部导师。新人可以反复问"这个流程为什么这样设计"而不用担心打扰同事。

**关键适配**：直接将公司 Wiki/Confluence 文档拆分为主题列表；CLAUDE.md 中设置"不确定时建议查阅内部文档"而非搜索；增加情景模拟——"如果遇到 X 情况，按流程你应该怎么做？"

## 核心原则

| 原则 | 说明 |
|------|------|
| **交互方式不变** | 苏格拉底对话 + 理解验证的核心机制，适用于任何知识领域 |
| **目录结构不变** | `sessions/` + `progress/` 的结构是通用的 |
| **追踪维度不变** | 已掌握、知识差距、信心水平覆盖所有学习场景 |
| **知识内容可替换** | 只需替换领域名称、模块划分、主题列表、权威来源 |
| **先规划后执行** | 填入框架前，花 30 分钟定义好自己的知识模块和主题清单 |
