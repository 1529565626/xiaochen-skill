# 通用引导式学习框架

> 苏格拉底式对话 + 结构化追踪 = AI 驱动的高效系统化学习

本项目提供一套**可复用的 AI 引导式学习方法论**。核心思想：交互方式是本质，知识内容可替换。只需替换领域名称和知识模块，即可用同一套机制学习任何系统性知识。

---

## 框架核心

```
学生提问 → AI 先问"你已知什么" → 基于已知构建新知识
         → 解释后追问验证 → 根据反馈调整教学
         → 自动记录会话 + 更新进度追踪
```

| 不变的部分（共性） | 可变的部分（替换） |
|---|---|
| 苏格拉底式教学对话模式 | 知识领域名称 |
| 会话记录模板与追踪结构 | 知识模块划分与主题列表 |
| /sessions + /progress 目录结构 | 学习资料引用与权威来源 |
| 验证与搜索协议 | 领域特定规则 |

### 认知科学依据

- **检索练习**：追问"你已知什么"强制大脑主动检索，增强记忆编码
- **精细加工**：将新知识与已有知识连接，形成更丰富的神经关联
- **间隔重复**：通过进度追踪器标记已掌握和薄弱点，自然形成间隔复习
- **即时反馈**：每次理解检查提供即时纠正，避免错误固化
- **元认知**：学生自己评估信心水平，提升对自身知识状态的觉察

```
└── /progress/             # 学习进度追踪（唯一数据源）
    └── 【领域】-study-tracker.md
```

## CLAUDE.md 核心要素

每个学习项目的 CLAUDE.md 定义以下内容：

1. **角色定义**：AI 扮演的导师角色和教学领域
2. **教学理念**：苏格拉底式提问法 + 主动验证 + 适应性跟进
3. **回复结构**：初步探索 → 讲解（约 200 词）→ 理解检查 → 适应性跟进
4. **知识体系**：按重要性/权重排序的模块与主题列表
5. **会话追踪协议**：两步流程——记录会话详情 + 更新进度追踪器
6. **强制验证协议**：不确定时必须在线搜索，绝不猜测

## 启动新学习项目的清单

1. 复制框架文件到新目录
2. 按顺序替换所有 `【】` 占位符
3. 将学习领域拆分为 5-10 个模块，按重要性排序
4. 每个模块下列出具体主题（总数建议 30-80 个）
5. 根据重要程度分配权重百分比
6. 创建空的进度追踪文件
7. 清空示例会话记录，保留模板
8. 打开 Claude Code，开始第一个问题

### 适用边界

- **适合**：有明确知识体系的学习（考试、认证、系统化学科）
- **不太适合**：纯技能训练（如乐器、体育）、完全开放的探索性学习
- **需要学生主动参与**：如果学生不愿回答追问，苏格拉底方法会失效

## 项目案例：AI-Prompt-Study

本仓库包含一个完整的实践案例——**AI 提示词工程系统化学习**（`/AI-Prompt-Study/`），展示了框架的实际应用。

### 案例概览

| 项目 | 说明 |
|---|---|
| **学习领域** | AI 提示词工程（Prompt Engineering） |
| **知识规模** | 7 大模块、41 个主题 |
| **导师角色** | AI 提示词工程互动式导师 |
| **学习目标** | 独立设计、优化和调试各类场景的提示词 |
| **参考来源** | Anthropic 官方文档、OpenAI Cookbook、arXiv 论文 |

### 知识体系（7 模块 · 41 主题）

| 模块 | 权重 | 主题数 | 核心内容 |
|---|---|---|---|
| A. 提示词基础理论 | 20% | 6 | LLM 原理、提示词要素、幻觉与局限、评估方法 |
| B. 提示词结构设计 | 20% | 6 | CRISPE 框架、输出格式控制、模板化设计 |
| C. 角色与上下文工程 | 15% | 5 | 角色设定、上下文管理、多轮对话状态 |
| D. 推理引导技术 | 15% | 7 | CoT、Few-shot、ReAct、思维树 |
| E. 提示词优化与调试 | 10% | 5 | 调试方法论、元提示词、自动优化 |
| F. 提示词安全与防护 | 10% | 5 | 注入防御、越狱防护、红队测试 |
| G. 实战场景应用 | 10% | 7 | 代码、写作、数据分析、多模态等场景 |

### 案例中的关键设计决策

**1. 知识模块按"理论→技能→安全"分层**

先建立正确的认知框架（A），再掌握核心工程技能（B/C/D），最后学习优化与安全（E/F），实战（G）贯穿全程。这比按字母顺序或随机排列更符合认知规律。

**2. 权重分配反映实践优先级**

提示词工程是实践学科，基础和结构设计各占 20%（共 40%），因为它们是所有高级技术的前提。安全与防护虽重要但初学者可后置（各 10%）。

**3. CLAUDE.md 中的教学示例**

框架要求在 CLAUDE.md 中预写互动示例，这确保了 AI 导师的行为一致性。AI-Prompt-Study 的示例以"怎么写 system prompt"为场景，展示了从"先问已知"到"解释三要素"到"理解检查"的完整对话流程。

**4. 双重进度追踪**

- `progress/ai-prompt-study-tracker.md`：细粒度追踪 41 个主题的掌握状态（框架标准要求）
- `PROGRESS.md`：高层级的课程任务清单（针对教学场景的补充，10 个任务关卡）

这种双层结构适用于"有人带领的系统课程"场景——PROGRESS.md 是课程大纲视角，tracker 是知识掌握视角。

**5. 会话模板的领域定制**

在通用 SESSION-TEMPLATE.md 基础上，增加了"提示词实践"字段（原始提示词 → 优化后提示词 → 优化要点），适应提示词工程"动手写"的学习特点。

### 案例文件结构

```
AI-Prompt-Study/
├── CLAUDE.md                         # AI 提示词工程导师行为指令
├── README.md                         # 项目说明与使用指南
├── PROGRESS.md                       # 课程任务清单（10 关）
├── progress/
│   └── ai-prompt-study-tracker.md   # 41 主题细粒度追踪
└── sessions/
    └── SESSION-TEMPLATE.md           # 领域定制的会话模板
```

## 如何使用本仓库

### 直接学习 AI 提示词工程

```bash
cd AI-Prompt-Study
claude-code
```

直接开始提问，Claude 会自动以苏格拉底方式引导你学习全部 41 个主题。

### 以此为模板启动你自己的学习项目

```bash
# 1. 复制框架
cp LEARNING-FRAMEWORK.md my-study/LEARNING-FRAMEWORK.md

# 2. 参考案例创建你的目录结构
# 3. 替换所有占位符，定义你的知识模块
# 4. 启动学习
cd my-study
claude-code
```
